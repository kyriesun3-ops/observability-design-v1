## 1. 背景与问题

OneLinkAI 是多资源 AI 平台，单个租户同时消费四类资源：**API 路由（按 token）、GPU 容器实例（按机时）、云电脑 Desk（按规格 + 时长）、云硬盘（按容量包月）**。

租户管理员当前缺乏一个统一视角回答三个问题：

- **钱花得值不值** —— 成本花在哪些资源 / 模型 / 用户上，有没有浪费（如挂机的云电脑、可被缓存挡掉的请求）。
- **用得好不好** —— 延迟、报错、缓存命中、风险拦截等质量与安全表现如何。
- **谁在用** —— 哪些用户 / 部门是重度消费方，便于成本分摊（showback）。

现有平台各资源的用量散落在各自模块，没有跨资源聚合、没有统一时间口径、没有可下钻的趋势分析。本模块即为填补这一空白。

**用户的"任务"（JTBD）**：作为租户管理员，当我每月 / 每周复盘账单时，我想在一个页面看清"钱花在哪、用得怎么样、谁在花"，以便我能优化成本、定位质量问题、向内部分摊费用。

> 框架说明：本节用 **Jobs-to-be-Done** 锚定真实任务（成本复盘 + 质量定位 + 费用分摊），而非从"做几个图表"出发，避免堆砌指标。

## 4. 全局设计铁律（贯穿所有子模块）

这是本模块区别于普通 dashboard 的核心约束，**所有指标实现必须遵守**：

### 4.1 一切跟随筛选窗口

页面顶部存在公共的「搜索 filter」与「时间筛选下拉」。**所有指标数值均为当前筛选窗口内的纯聚合值**，不得隐含固定时间基准（如"本月""今日"）。

- **不提供固定周期环比**：平台未定义统一统计周期，任何固定环比基准都会与任意时间窗冲突。趋势变化**只由折线图本身体现**。
- 例外：标记为「存量快照」的指标（如账户可用额度）为基线参照，不随时间窗收缩。

### 4.2 时间窗摊销成本（amortized cost）

所有成本类指标（KPI 总成本、成本构成、各模块成本卡）跟随时间筛选，展示"归属于所选窗口"的成本，分两类处理：

- **用量型**（API 按 token、GPU 实例按机时、Desk 按运行时长）：直接累加窗口内实际发生费用。
- **订阅 / 固定型**（云硬盘按容量包月、Desk 包月套餐）：按**日费率摊销** = `月费 ÷ 当月天数 × 窗口内天数`。

效果：无论选 24 小时还是 90 天，所有成本数字**可加、自洽**，占比图始终反映窗口内真实构成。

### 4.3 全局搜索筛选维度

参考 Portkey 并按本平台多资源特性扩展：

`Resource(资源类型) · Model · Cost · Tokens · Cache · Status · User · API Key · Provider · Trace Id`

> `Resource`（云电脑 / GPU 容器实例 / API 路由 / 云硬盘）是本平台新增的核心维度，Portkey 没有。

**筛选交互**：可叠加 chip 式筛选，一次加一个维度；选维度 → 填/选值 → 加为 chip，可逐个移除。`Model` 维度用于驱动各指标卡的"按模型"联动下钻。

## 6. 通用组件规范

| 组件 | 说明 | 关键行为 |
|------|------|---------|
| TimeFilter 时间筛选 | 预设窗口 + 自定义区间 | 选中值驱动各卡副标题 `数据来自 xx 至 xx` |
| FilterBar 全局筛选 | chip 叠加式多维筛选 | 选维度→填值→加 chip；× 删除 |
| 指标卡 XCard | 统一卡片容器（标题 + 数值 + tip + 副标题 + 图表） | 副标题显示当前时间窗；悬浮标题看口径 tip 与涉及模型 |
| Seg 分段切换 | 卡内文本式分段控件 | 用于「按模态趋势/模型排行」「总览/按模型」「模态标签」等卡内视图切换 |
| CustomTooltip | 图表统一 hover 提示 | 按 unit 格式化数值 |

**数值格式**：金额 `¥` + 千分位两位小数；大数 token 用 `M`（百万）；延迟/时长 `s`/`ms`；占比 `%`；调用量 `次`。

## 7. 子模块功能需求

> 各指标的精确公式见 `docs/metrics.md` 对应章节，此处给出产品定义、布局与交互重点。

### 7.1 消耗分析（Cost / Spend）

**目标**：回答"总共花了多少、花在哪、谁花的"。

- 顶部 KPI（存量快照）：可用额度（= 累计充值 + 赠金 − 累计消费）、累计充值、累计消费、赠金。
- 窗口聚合概览：总请求数、总 token（输入 + 缓存 + 输出拆分）、总消费（摊销口径）。
- 按服务商分布、按模型分布：费用 ¥ / token 个 可切换。
- 消耗排行榜：按 成本 / API Key / 用户 排序，定位重度消费方。

### 7.2 缓存命中（Cache）

**目标**：量化缓存省了多少钱、提了多少速。命中数（精确 + 语义）、命中率、加速比、节省成本、命中平均延迟。

### 7.3 报错分析（Errors）

**目标**：暴露失败请求与挽救情况。报错率、报错数（按 error code 堆叠）、错误类型种数、被挽救请求数。

### 7.4 延迟分析（Latency）

**目标**：定位慢在哪。端到端延迟（P50 / P95 / P99 切换）、首字延迟 TTFT、延迟类别分布（网络 / 推理 / 排队三段堆叠）。

### 7.5 多媒体模型（Multimodal）— 本轮迭代重点

**目标**：跨模态（图 / 音 / 视频）聚合用量与成本，并清晰呈现总调用量、模态排行与生成时长。

**计量归一化**：图 = 张、音频 = 分钟、视频 = 秒，单页聚合时统一折算为「次 / $」，原生单位下钻留 V2。

**同步/异步分治**（影响口径）：同步·理解类（文+图 / 文+视频 / 音+文）→ 输出为 token，复用 LLM 路由分析范式；异步·生成处理类（纯视频生成 / TTS / STT）→ 核心为处理时长。

**卡片布局（4 张）**：

1. **多模态调用量（整合卡 · 调用量 + 模型排行）** ★本轮整合
   - 卡片数值 = 窗口内多模态总调用次数（如 `12,480 次`）。
   - 卡内 `Seg` 切换两种视图：
     - **按模态趋势**：图像 / 音频 / 视频 按模态堆叠的柱状趋势图。
     - **模型排行**：跨模态生成模型按调用量降序的横条榜（Top 6，颜色按模态区分，附模态图例）。
   - 设计动机：原「多模态调用量」与「多模态模型调用排行」两张分离卡片信息割裂；整合后一张卡即可看清**总量 → 模态构成 → 具体模型排名**的递进关系。

2. **生成类型分布**（保留）：图片 / 视频 / 音频 产物数量占比环形图。

3. **多模态成本**（保留）：按模态（图像 / 音频 / 视频）归因的消费金额堆叠，视频单价高需重点观察。

4. **平均生成时长（整合卡 · 模态标签 + 总览/按模型）** ★本轮整合
   - 把原图像 / 视频 / 音频三张独立的「平均生成时长」卡合并为一张。
   - 顶部 `Seg` **模态标签切换**：图像 | 视频 | 音频，**默认视频**（视频生成最慢、最值得关注）。
   - 右侧 **总览 / 按模型** 切换 + **P50 / P95 / P99** 分位选择：
     - **总览**：所选模态的处理时长单条折线。
     - **按模型**：所选模态下各生成模型的处理时长多条折线（如视频：sora / kling / runway…）。
   - 设计动机：三张分卡占位且对比割裂；合并后默认聚焦视频、一处切换即可横比模态与模型，信息密度与可比性更高。

### 7.6 风险拦截（Risk）

**目标**：确认安全策略生效、定位高风险用户。拦截数、拦截率、风险类型分布、Top 风险用户榜、拦截记录明细。

### 7.7 云电脑 Desk

**目标**：成本可见 + 揪出挂机浪费。计费按规格 + 时长；存在「租户分配 / 用户自购」双开通路径，故**状态类指标一律按桌面真实系统状态统计**，不依赖开通来源。

- Desk 总成本（按规格）★hero（摊销口径）、资源利用率（CPU/内存/磁盘/GPU）、桌面状态分布（运行中/已关机）、连接质量 RTT（P50/P95，>200ms 体验下降）、桌面使用明细（利用率 < 20% 标红，含「开通来源」列）。

### 7.8 用户使用（Users · 全平台聚合）

**目标**：成本分摊（showback）+ 识别重度用户。覆盖 云电脑 + GPU 容器实例 + API 路由 + 云硬盘 四类资源。

- 活跃用户数（窗口内有任意计量行为的去重用户）、全平台总成本（四资源摊销合计）、人均成本、成本构成（按资源类型环形图）、用户使用趋势（Unique Users & Request per User 双轴）、用户请求列表（含部门列为 V2 铺路）。

## 8. 关键交互流程

1. **调时间窗口** → 全部指标卡重新聚合；成本卡按摊销口径重算；副标题更新为新窗口。
2. **加筛选 chip** → 命中数据重算；`Model` 维度联动各卡"按模型"。
3. **多模态调用卡** → `按模态趋势 ⇄ 模型排行` 卡内切换。
4. **平均生成时长卡** → 模态标签（默认视频）切换 + `总览 ⇄ 按模型` + 分位切换，三者组合即时重算曲线。
5. **切 Tab** → 切换分析视角，筛选与时间窗口保持不变（共享顶部条件）。

## 11. 风险评估（Cagan 四类风险）

- **价值风险**：成本可见性与浪费定位是租户管理员真实高频痛点，方向明确。摊销成本对用户的直观性需配 tip 解释。
- **可用性风险**：多模态整合卡引入卡内多重切换（模态 / 总览-按模型 / 分位），需可用性测试确认控件组合不致混淆；取消固定环比后用户须能从折线读出趋势。
- **可行性风险**：摊销成本与四资源聚合依赖后端账单/计量打通，是落地最大不确定项；Desk 状态枚举依赖外部系统；多模态「按模型」需后端提供按生成模型的处理时长序列。
- **商业可行性**：服务存量租户留存与扩容，定位为平台增值能力，不直接产生收入。
